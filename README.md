# Trend to Video Studio

An automation-first content pipeline for:
- discovering trending video topics
- extracting reusable structure from public examples
- generating original prompts/storyboards
- producing original AI videos
- preparing uploads for YouTube scheduling

## Scope
This project is designed for original content generation and publishing workflows.
It does **not** implement watermark removal, DRM bypass, or direct cloning of protected media.

## MVP Modules
1. Trend discovery
2. Source analysis
3. Screenshot / keyframe extraction
4. Prompt generation
5. AI video generation adapter
6. Asset packaging
7. YouTube upload + scheduling
8. Job orchestration dashboard

## Stack (planned)
- Next.js / TypeScript
- Node job runners
- Supabase or SQLite/Postgres for jobs + metadata
- YouTube Data API
- Pluggable video generation providers

## Data Storage
Default repository driver:
- `json`

Default snapshot paths:
- SQLite: `.data/project-snapshot.sqlite`
- JSON fallback: `.data/project-snapshot.json`

Override with environment variables:
- `TREND_TO_VIDEO_REPOSITORY_DRIVER=sqlite|json`
- `TREND_TO_VIDEO_SQLITE_FILE=/absolute/path/to/project-snapshot.sqlite`
- `TREND_TO_VIDEO_DATA_FILE=/absolute/path/to/project-snapshot.json`

When `sqlite` is active, the app auto-migrates the existing JSON snapshot into SQLite on first read if a SQLite snapshot does not exist yet.

## Real Generation / Upload Integration Contract
Copy `.env.example` to your local env file and set these values.

### Video generation webhook/API
Environment variables:
- `TREND_TO_VIDEO_GENERATION_ENDPOINT`
- `TREND_TO_VIDEO_GENERATION_API_KEY` (optional)
- `TREND_TO_VIDEO_DEFAULT_VIDEO_PROVIDER=env-webhook-video` (optional)

Request body sent by the app:
```json
{
  "prompt": "Create an original short-form video ..."
}
```

Expected JSON response:
```json
{
  "jobId": "gen_123",
  "status": "completed",
  "outputUrl": "https://cdn.example.com/output.mp4"
}
```

Accepted response fields:
- `jobId` (optional)
- `status` = `queued` or `completed` (optional)
- `outputUrl` or `videoUrl`

### Upload webhook/API
Environment variables:
- `TREND_TO_VIDEO_UPLOAD_ENDPOINT`
- `TREND_TO_VIDEO_UPLOAD_API_KEY` (optional)
- `TREND_TO_VIDEO_DEFAULT_UPLOAD_PROVIDER=env-webhook-upload` (optional)

Request body sent by the app:
```json
{
  "videoUrl": "https://cdn.example.com/output.mp4",
  "title": "Trend Prompt Draft",
  "description": "Generated by trend-to-video-studio",
  "scheduledFor": "2026-03-13T12:00:00.000Z",
  "platform": "youtube"
}
```

Expected JSON response:
```json
{
  "jobId": "upload_123",
  "status": "completed",
  "remoteUrl": "https://youtube.com/watch?v=example"
}
```

Accepted response fields:
- `jobId` (optional)
- `status` = `queued` or `completed` (optional)
- `remoteUrl` or `uploadUrl`

## Minimal Smoke Path
Once env is configured, you can run the smallest real integration path like this:

1. Start with an existing trend candidate and generated prompt draft in the local repository.
2. Hit the video job API with your real generation provider:
```bash
curl -X POST http://localhost:3000/api/video-jobs \
  -H "Content-Type: application/json" \
  -d '{
    "promptDraftId": "prompt_...",
    "provider": "env-webhook-video"
  }'
```
3. Take the returned `videoJobId` / generated job item and call upload:
```bash
curl -X POST http://localhost:3000/api/upload-jobs \
  -H "Content-Type: application/json" \
  -d '{
    "videoJobId": "video_...",
    "provider": "env-webhook-upload",
    "title": "Trend-to-Video Smoke Test",
    "description": "Real upload integration smoke test"
  }'
```
4. Or run the worker end-to-end with env-driven providers:
```bash
pnpm dev:worker -- --mode process-one --candidate-id trend_... --provider env-webhook-video --upload-provider env-webhook-upload
```

## API Notes
`POST /api/trend-candidates` accepts:
- `topic`
- `sourceUrl`
- `sourcePlatform` (`youtube` | `shorts` | `manual`)

For non-manual inputs, the backend validates and normalizes supported YouTube URLs before persistence.
